## 3. 🤔 C 语言中有哪些类型的符号？

从编译器的视角看，源代码会被拆分成一串符号，也就是记号，主要可以分成几大类：

- 标识符：用来给变量、函数、类型等命名。
- 关键字：语言保留的具有特殊语义的单词。
- 常量和字面量：如整数字面量、浮点字面量、字符常量、字符串字面量等。
- 运算符：用来表示各种运算和关系。
- 分隔符与界定符：如分号、逗号、括号、花括号等。
- 预处理相关符号：主要是行首的 `#` 号，以及宏定义中使用的部分符号。

理解这些符号的分类，有助于更好地阅读编译器报错信息，也有助于在阅读标准时对号入座。

## 4. 🤔 什么是标识符？命名规则是什么？

标识符是程序中给实体起的名字，实体可以是变量、函数、类型、结构体枚举标签、宏名等。

C 语言中标识符的命名规则大致是：

- 只能由字母、数字和下划线组成。
- 第一个字符不能是数字，一般以字母或下划线开头。
- 区分大小写，`count` 和 `Count` 被视为两个不同的标识符。
- 不能与关键字同名，否则会被当作关键字处理。

在可移植的程序中，还应避免使用以下形式：

- 以双下划线开头，或以下划线加大写字母开头，这类名字通常保留给实现使用。
- 与常见库函数名过于接近，以免造成歧义。

良好的命名能显著提升代码可读性，比如选择能表达含义的英文单词，而不是随意的缩写。

## 5. 🤔 关键字与标识符有何区别？

关键字是由语言标准预先保留的一组单词，用来表达固定的语法结构，例如 `if`、`for`、`int` 等。

关键字与普通标识符的主要区别是：

- 语义地位不同：
  - 关键字直接参与语法结构的构成。
  - 标识符只是为某个实体贴上的名字。
- 可否自定义：
  - 关键字集合由标准规定，程序员不能新增或重定义。
  - 标识符由程序员自由命名，只要不与关键字冲突即可。
- 解析方式：
  - 词法分析器会优先识别关键字。
  - 如果一个单词不在关键字列表中，就会被当作标识符处理。

标准库中的函数名如 `printf` 不属于关键字，只是普通的标识符，只是这些名字在库中已经有约定的含义。

## 6. 🤔 运算符属于符号吗？如何分类？

运算符属于 C 语言符号体系中的重要一类，多数运算符由一个或多个标点字符组成。

从作用上可以把常见运算符大致分成几组：

- 算术运算符：加、减、乘、除、取模。
- 关系运算符：大于、小于、大于等于、小于等于、等于、不等于。
- 逻辑运算符：与、或、非。
- 位运算符：按位与、按位或、按位异或、移位。
- 赋值与复合赋值运算符，以及自增、自减。
- 其他运算符：条件运算符、函数调用运算符、下标运算符、成员访问运算符等。

从元数上看，还可以分成一元运算符，如取反、自增、自减；二元运算符；以及唯一的三元运算符。

## 7. 🤔 分隔符（如 ; , {} ()）的作用是什么？

分隔符和界定符主要用来划分和组织程序结构，虽然本身不参与数值运算，却对语法至关重要。

- 分号：
  - 标记一条简单语句的结束。
  - 在很多情况下，少写一个分号就会导致语法错误。
- 逗号：
  - 在一条语句中分隔多个声明或多个实参。
  - 需要注意逗号运算符和参数分隔逗号在语义上的不同。
- 圆括号：
  - 用来改变表达式的结合顺序。
  - 用在函数声明和调用中表示参数列表。
- 花括号：
  - 用来构成复合语句，也就是作用域块。
  - 控制变量的可见范围和生命周期。

熟练掌握这些分隔符的用法，有助于写出结构清晰且不易出错的 C 程序。

## 8. 🤔 预处理指令中的 # 是什么性质的符号？

预处理指令是在正式编译之前由预处理器处理的一类命令，例如 `include`、`define` 等。

在源文件中，预处理指令通常以行首的 `#` 号开头，其作用特点是：

- 由预处理器在编译前先行处理，例如展开宏、替换头文件、条件编译等。
- 这一阶段发生的文本替换，会影响后续编译器看到的真正源代码内容。
- `#` 号本身并不出现在最终的目标程序中，它只在翻译阶段起作用。

可以把预处理指令看成是对源代码进行的一层文本级重写工具，与普通语句属于不同的语法层次。

## 9. 🆚 字符常量 'a' 与字符串字面量 "a" 在符号层面有何不同？

虽然 `'a'` 和 `"a"` 在源码中看起来很接近，但它们在类型和存储方式上有本质区别：

| 特性 | 字符常量 `'a'` | 字符串字面量 `"a"` |
| --- | --- | --- |
| 含义 | 单个字符的数值表示 | 含有结尾空字符的字符数组 |
| 类型 | 整型或 `char`，取决于实现 | `char` 数组类型 |
| 存储大小 | 占用一个字符宽度 | 至少两个字符，一个是内容，一个是结束符 |
| 典型用途 | 参与比较、赋值给字符变量 | 初始化字符串、指针指向字符串常量 |

理解这一区别，对于正确处理字符串长度、数组大小以及指针运算都非常重要。

## 10. 🤔 空格、制表符、换行符在 C 语言中算符号吗？

从源代码文本的角度看，空格、制表符、换行符等都属于字符，也可以被视为符号，但在语法层面它们通常被统称为空白符。

空白符的主要作用是分隔记号，让编译器能够正确识别单词边界，例如把两个标识符分开。

在大多数情况下，多个空白符等价于一个空白符，编译器会直接忽略，但有一些例外：

- 字符常量和字符串字面量内部的空白符是内容的一部分。
- 预处理指令行中的换行，对宏定义和条件编译有严格影响。
- 某些旧式编译器对行长度和缩进还有历史遗留限制。

从可读性角度看，合理使用空格和缩进远比机械地压缩空白更重要，推荐遵循项目统一的代码风格约定。
